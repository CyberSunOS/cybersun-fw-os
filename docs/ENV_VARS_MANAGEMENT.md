# Environment Variables Management

**Purpose**: Ensure AI agents track and communicate all required environment variables  
**Version**: 1.0.0

---

## ⚠️ CRITICAL: Track ALL Environment Variables

When building an application, AI agents MUST:

1. **Track every env var** needed for the app to work
2. **Ask the user** for values they need to provide
3. **Document in project** which vars are required vs optional
4. **Never assume** default values for secrets/credentials

---

## 📋 ENV_VARS.md Template

Create this file in every project root:

```markdown
# Environment Variables

## Required (App will NOT work without these)

| Variable | Purpose | Who Provides | Example |
|----------|---------|--------------|---------|
| `DATABASE_URL` | PostgreSQL connection | User | `postgresql://...` |
| `NEXTAUTH_SECRET` | Session encryption | User (generate) | `openssl rand -base64 32` |
| `OPENAI_API_KEY` | AI functionality | User | `sk-...` |

## Optional (Has defaults, but user can override)

| Variable | Purpose | Default | Override When |
|----------|---------|---------|---------------|
| `NODE_ENV` | Environment mode | `development` | Deploy to production |
| `PORT` | Server port | `3000` | Port conflict |

## Auto-Generated (Don't ask user for these)

| Variable | Purpose | Generated By |
|----------|---------|--------------|
| `NEXT_PUBLIC_APP_URL` | Public URL | Build process |

## Setup Commands

```bash
# Generate NEXTAUTH_SECRET
openssl rand -base64 32

# Copy template
cp .env.example .env

# Edit with your values
nano .env
```
```

## 🔄 AI Agent Workflow

### During Development

```
1. When adding a feature that needs an env var:
   - Add to ENV_VARS.md immediately
   - Add to .env.example with placeholder
   - Inform user what they need to provide

2. Before any deployment:
   - Review ENV_VARS.md
   - Ask user: "Do you have values for: X, Y, Z?"
   - Provide generation commands where applicable
```

### Example Conversation Flow

```
AI: "This feature requires an OpenAI API key.

I've added OPENAI_API_KEY to:
- ENV_VARS.md (documented)
- .env.example (placeholder)

Do you have an OpenAI API key, or do you need help getting one?"

User: "I have one"

AI: "Great! Please add it to your .env file:
OPENAI_API_KEY=your-key-here

I won't see or store your key - just add it locally."
```

## 📁 Required Project Files

### .env.example (commit to git)

```bash
# Database
DATABASE_URL=postgresql://user:password@localhost:5432/dbname

# Authentication  
NEXTAUTH_SECRET=generate-with-openssl-rand-base64-32
NEXTAUTH_URL=http://localhost:3000

# External APIs (get from provider)
OPENAI_API_KEY=sk-your-key-here

# Optional
NODE_ENV=development
```

### .env (NEVER commit - add to .gitignore)

Actual values go here.

### .gitignore (ensure .env is ignored)

```
.env
.env.local
.env.*.local
```

## 🏷️ Environment Variable Categories

### 1. Secrets (User MUST provide)
- API keys
- Database passwords
- OAuth secrets
- Encryption keys

### 2. Configuration (User provides for their setup)
- URLs/domains
- Email addresses
- Project IDs

### 3. Build-time (Set during deployment)
- `NODE_ENV`
- `NEXT_PUBLIC_*` vars

### 4. Auto-generated (App/framework generates)
- Session tokens
- Cache keys
- Build hashes

---

## 🚀 Deployment Checklist

Before deploying, ensure user has:

- [ ] All "Required" vars from ENV_VARS.md
- [ ] Correct values for their environment (prod vs dev)
- [ ] Secrets stored securely (not in git)
- [ ] Platform-specific vars set (Coolify, Vercel, etc.)

### Platform-Specific Notes

**Coolify**:
- Set env vars in Dashboard → Environment Variables
- Use "Build" vs "Runtime" appropriately

**Vercel**:
- Set in Project Settings → Environment Variables
- Choose correct environment (Production/Preview/Development)

**Docker**:
- Use `--env-file .env` or `-e VAR=value`
- Never bake secrets into images

## ⚡ Quick Reference

### When to Ask User for Values

| Situation | Ask User? |
|-----------|-----------|
| Database connection string | ✅ Yes |
| API keys for external services | ✅ Yes |
| OAuth client secrets | ✅ Yes |
| `NODE_ENV` | ❌ No (use default) |
| App port | ❌ No (use default) |
| Session secret | ✅ Yes (provide generation command) |

### Generation Commands

```bash
# Random secret (32 bytes, base64)
openssl rand -base64 32

# UUID
uuidgen

# Random hex string
openssl rand -hex 16
```
